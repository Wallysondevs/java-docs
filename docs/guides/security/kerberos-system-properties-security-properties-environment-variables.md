# Propriedades de Sistema, Propriedades de Segurança e Variáveis de Ambiente Kerberos

## Propriedades de Sistema, Propriedades de Segurança e Variáveis de Ambiente Kerberos

A tabela a seguir lista as propriedades de sistema, propriedades de segurança e variáveis de ambiente relacionadas ao Kerberos.

Nota:

Algumas propriedades são tanto uma propriedade de segurança quanto uma propriedade de sistema. Se você definir um valor para a propriedade de sistema, ele substituirá qualquer valor definido como uma propriedade de segurança. Consulte [Como Especificar uma Propriedade java.lang.System](<#/doc/guides/security/java-secure-socket-extension-jsse-reference-guide>) e [Como Especificar uma Propriedade java.security.Security](<#/doc/guides/security/java-secure-socket-extension-jsse-reference-guide>).

Tabela 7-2 Propriedades e Variáveis de Ambiente Kerberos

Nome | Tipo | Descrição | Valor Padrão
---|---|---|---
`KRB5CCNAME` | Variável de ambiente | Especifica o nome do caminho completo para o arquivo Kerberos credentials cache (CC). | O valor padrão depende do sistema operacional. Para Linux e macOS, é `/tmp/krb5cc_userid`
`LOGONSERVER` | Variável de ambiente | No Windows, o KDC padrão se um arquivo `krb5.conf` não puder ser encontrado ou se ele não contiver configurações para o KDC do realm padrão. Consulte [Localizando o Arquivo de Configuração krb5.conf](<#/doc/guides/security/kerberos-requirements>) | Consulte [Localizando o Arquivo de Configuração krb5.conf](<#/doc/guides/security/kerberos-requirements>)
`USERDNSDOMAIN` | Variável de ambiente | No Windows, o realm padrão se um arquivo `krb5.conf` não puder ser encontrado ou se ele não contiver configurações para o realm padrão. Consulte [Localizando o Arquivo de Configuração krb5.conf](<#/doc/guides/security/kerberos-requirements>). | Consulte [Localizando o Arquivo de Configuração krb5.conf](<#/doc/guides/security/kerberos-requirements>)
`jdk.security.krb5.default.initiate.credential` | Propriedade de segurança e propriedade de sistema | Especifica a política para a entrada de configuração `proxy_impersonator` do Kerberos ccache. A entrada de configuração `proxy_impersonator` do ccache indica que o ccache é uma credencial delegada sintética para uso com a extensão Kerberos Service for User to Proxy ([S4U2proxy](<https://learn.microsoft.com/en-us/openspecs/windows_protocols/ms-sfu/bde93b0e-f3c9-4ddf-9f44-e1453be7af5a>)) por um servidor intermediário. O arquivo ccache também deve conter o Ticket Granting Ticket (TGT) deste servidor e um ticket de evidência do principal padrão do ccache para este servidor. O valor desta propriedade de segurança determina como o Java usa esta entrada de configuração. Pode ter um dos seguintes valores:

  * `no-impersonate`: Ignora esta entrada de configuração e sempre age como o proprietário do TGT (se existir).
  * `try-impersonate`: Tenta a impersonation quando esta entrada de configuração existe. Se nenhum TGT ou ticket de evidência correspondente for encontrado, retorna para `no-impersonate`.
  * `always-impersonate`: Sempre faz a impersonation quando esta entrada de configuração existe. Se nenhum TGT ou ticket de evidência correspondente for encontrado, nenhuma credencial inicial é lida do ccache.

| `always-impersonate`
`jdk.security.krb5.name.case.sensitive` | Propriedade de segurança e propriedade de sistema | Se `true`, então a comparação de nomes de principal durante a pesquisa de entradas de keytab e credentials cache (ccache) diferencia maiúsculas de minúsculas. O valor padrão é `false` para garantir a compatibilidade retroativa. Ao procurar uma entrada de keytab ou ccache para um principal Kerberos, o nome do principal é comparado com o nome da entrada de forma não sensível a maiúsculas e minúsculas. No entanto, muitas implementações Kerberos tratam os nomes de principal como sensíveis a maiúsculas e minúsculas. Como resultado, se dois principais tiverem nomes que diferem apenas por maiúsculas e minúsculas, há o risco de selecionar a entrada de keytab ou ccache incorreta. | `false`
`jdk.security.krb5.s4u2proxy.acceptNonForwardableServiceTicket` | Propriedade de segurança e propriedade de sistema | Especifica a política para um service ticket não forwardable em uma requisição S4U2proxy. A extensão Kerberos Service for User to Proxy ([S4U2proxy](<https://learn.microsoft.com/en-us/openspecs/windows_protocols/ms-sfu/bde93b0e-f3c9-4ddf-9f44-e1453be7af5a>)) permite que um serviço intermediário obtenha um service ticket para outro serviço em nome de um usuário. Isso exige que o service ticket do usuário para o primeiro serviço tenha a flag `forwardable` definida. No entanto, algumas implementações de KDC ignoram este requisito e aceitam service tickets com a flag desativada. Se esta propriedade de segurança for definida como `true`, então:

  * O service ticket do usuário, quando obtido pelo serviço intermediário após uma impersonation S4U2self, não é obrigado a ter a flag `forwardable`.
  * Se uma requisição S4U2proxy receber um KRB_ERROR com o código de erro KDC_ERR_BADOPTION e o ticket para o serviço intermediário não for `forwardable`, então o JDK tentará a mesma requisição com outro KDC em vez de tratá-la como uma falha fatal.

| `false`
`krb5.kdc.bad.policy` | Propriedade de segurança e propriedade de sistema | Especifica a política para falhas nas pesquisas de KDC Kerberos. Quando um KDC está indisponível (devido, por exemplo, a um erro de rede ou falha de serviço), ele é colocado em uma lista secundária e acessado com menos frequência para futuras requisições. O valor desta propriedade de segurança pode ser um dos seguintes:

  * `tryLast`: KDCs na lista secundária são sempre tentados depois daqueles que não estão na lista. Exemplo: `krb5.kdc.bad.policy = tryLast`
  * `tryLess[:max_retries,timeout]`: KDCs na lista secundária ainda são tentados em sua ordem na configuração, mas com valores menores de `max_retries` e `timeout`. `max_retries` e `timeout` são parâmetros numéricos opcionais (padrão 1 e 5000, o que significa uma vez e 5 segundos). Observe que se qualquer um dos valores definidos aqui for maior do que o definido em `krb5.conf`, eles serão ignorados. Exemplo: `krb5.kdc.bad.policy = tryLess:2,2000`

Sempre que um KDC é detectado como disponível, ele é removido da lista secundária. A lista secundária é redefinida quando `krb5.conf` é recarregado. Você pode adicionar `refreshKrb5Config=true` a um arquivo de configuração JAAS para que `krb5.conf` seja recarregado sempre que uma autenticação JAAS for tentada. | `tryLast`
`sun.security.krb5.disableReferrals` | Propriedade de segurança e propriedade de sistema | Desabilita referências cross-realm do Kerberos. O cliente Kerberos do JDK suporta referências cross-realm conforme definido na [RFC 6806: Kerberos Principal Name Canonicalization and Cross-Realm Referrals](<https://datatracker.ietf.org/doc/html/rfc6806>). Isso permite configurar ambientes mais dinâmicos nos quais os clientes não precisam saber antecipadamente como alcançar o realm de um principal alvo (seja um usuário ou serviço). Quando um cliente emite uma requisição Kerberos Authentication Service (AS) ou Ticket-Granting Service (TGS), a opção "canonicalize" é definida para anunciar o suporte a este recurso. Um servidor KDC pode atender à requisição ou responder referindo o cliente a um diferente. Se referido, o cliente emitirá uma nova requisição e o ciclo se repetirá. Além das referências, a opção "canonicalize" permite que o servidor KDC altere o nome do cliente em resposta a uma requisição AS. Por razões de segurança, o esquema FAST é imposto. Consulte RFC 6806, seção 11, "Negotiation of FAST and Detecting Modified Requests." | `false`
`sun.security.krb5.maxReferrals` | Propriedade de segurança e propriedade de sistema | Número máximo de referências AS ou TGS para evitar loops infinitos. | `5`
`java.security.krb5.conf` | Propriedade de sistema | Especifica a localização do arquivo `krb5.conf`. Consulte [Localizando o Arquivo de Configuração krb5.conf](<#/doc/guides/security/kerberos-requirements>) para mais informações. | Nenhum
`java.security.krb5.kdc` | Propriedade de sistema | Especifica o KDC padrão para o realm padrão especificado por `java.security.krb5.realm`. Consulte [Definindo Propriedades para Indicar o Realm e KDC Padrão](<#/doc/guides/security/kerberos-requirements>) para mais informações. | Nenhum
`java.security.krb5.realm` | Propriedade de sistema | Especifica o realm padrão. Consulte [Definindo Propriedades para Indicar o Realm e KDC Padrão](<#/doc/guides/security/kerberos-requirements>) para mais informações. | Nenhum
