# Acessando Recursos usando Paths

## Apresentando a Classe File Legada

Existem duas maneiras de modelar um arquivo ou um path em um file system em Java. A primeira, legada, é a classe [`File`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html>). Esta classe é mencionada aqui, com uma advertência: você não deve mais usá-la em seu código, a menos que tenha ótimas razões para fazê-lo. Você deve favorecer o uso da interface [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>), também abordada nesta seção. Juntamente com os métodos de fábrica da classe [`Files`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Files.html>), ela oferece mais recursos do que a classe [`File`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html>), e melhor desempenho, especialmente para acessar arquivos e diretórios maiores.

Dito isso, por ser amplamente utilizada em código legado, entender a classe [`File`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html>) ainda pode ser importante para você. Sem nos aprofundarmos muito, apresentaremos os principais conceitos desta classe.

Uma instância da classe [`File`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html>) pode representar qualquer coisa em um file system: um arquivo, um diretório, um link simbólico, um path relativo ou um path absoluto. Esta instância é uma noção abstrata. Criar tal instância não cria nada em seu file system. Você pode consultar seu file system usando esta classe, mas precisa fazê-lo explicitamente.

Uma instância de [`File`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html>) não permite que você acesse o conteúdo do arquivo que ela representa. Com esta instância, você pode verificar se este arquivo existe ou é legível (entre outras coisas).

Um arquivo é composto por vários elementos, separados por um separador, que depende do seu file system. O primeiro elemento pode ser um prefixo, como um especificador de unidade de disco, uma barra para o diretório raiz do UNIX. Os outros elementos são nomes.

### Criando uma Instância de File

Você pode criar uma instância da classe [`File`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html>) usando vários construtores:

  * [`File(String pathName)`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#%3Cinit%3E\(java.lang.String\)>): cria um arquivo a partir do path que você fornece.
  * [`File(String parent, String child)`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#%3Cinit%3E\(java.lang.String,java.lang.String\)>): cria o arquivo dado no diretório `parent`.
  * [`File(File parent, String child)`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#%3Cinit%3E\(java.io.File,java.lang.String\)>): cria o arquivo dado no diretório `parent`, especificado como uma instância de [`File`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html>).
  * [`File(URI uri)`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#%3Cinit%3E\(java.net.URI\)>): cria um arquivo a partir de uma `URI`.

### Obtendo os Elementos de um Arquivo

Os seguintes métodos fornecem informações sobre os elementos deste arquivo:

  * [`getName()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#getName\(\)>): retorna o nome do arquivo ou diretório denotado por este objeto file. Este é apenas o último nome da sequência.
  * [`getParent()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#getParent\(\)>): retorna a string pathname do pai deste pathname abstrato, ou null se este pathname não nomear um diretório pai.
  * [`getPath()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#getPath\(\)>): retorna este pathname abstrato convertido em uma string pathname. Este método não está relacionado à interface [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>).
  * [`getAbsolutePath()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#getAbsolutePath\(\)>): retorna a string pathname absoluta deste pathname abstrato. Se este pathname abstrato já for absoluto, a string pathname é simplesmente retornada. Caso contrário, este pathname é resolvido de uma maneira dependente do sistema.
  * [`getCanonicalPath()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#getCanonicalPath\(\)>): retorna a string pathname canônica deste pathname abstrato. O pathname canônico é absoluto, único e dependente do sistema. A construção deste pathname canônico geralmente envolve a remoção de nomes redundantes como `.` e `..` do pathname, e a resolução de links simbólicos.

### Obtendo Informações sobre um Arquivo ou um Diretório

Alguns desses métodos podem exigir direitos especiais em arquivos ou diretórios. Como uma classe legada, a [`File`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html>) não expõe todos os atributos de segurança oferecidos pelo seu file system.

  * [`isFile()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#isFile\(\)>), [`isDirectory()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#isDirectory\(\)>): verifica se este pathname abstrato denota um arquivo ou diretório existente.
  * [`exists()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#exists\(\)>), [`canRead()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#canRead\(\)>), [`canWrite()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#canWrite\(\)>), [`canExecute()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#canExecute\(\)>): verifica se este arquivo existe, é legível, se você pode modificá-lo ou se pode executá-lo.
  * [`setReadable(boolean)`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#setReadable\(boolean\)>), [`setWritable(boolean)`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#setWritable\(boolean\)>), [`setExecutable(boolean)`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#setExecutable\(boolean\)>): permite que você altere o atributo de segurança correspondente do arquivo para o usuário que executa o programa. Esses métodos retornam `true` se a operação foi bem-sucedida.
  * [`setPosixFilePermissions(path,permissions)`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Files.html#setPosixFilePermissions\(java.nio.file.Path,java.util.Set\)>): permite que você atualize as permissões POSIX de um dado path. Este método recebe um conjunto de [`PosixFilePermissions`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/attribute/PosixFilePermissions.html>) como argumento. Este método retorna o [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>), permitindo encadear chamadas de método neste objeto. Obviamente, o file system ao qual este path pertence precisa suportar o [`PosixFileAttributeView`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/attribute/PosixFileAttributeView.html>). Se não for o caso, uma [`UnsupportedOperationException`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/lang/UnsupportedOperationException.html>) é lançada.
  * [`lastModified()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#lastModified\(\)>) e [`setLastModified()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#setLastModified\(long\)>): retornam ou definem a hora em que este arquivo foi modificado pela última vez.
  * [`length()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#length\(\)>): retorna o comprimento do arquivo denotado por este pathname abstrato.
  * [`isHidden()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#isHidden\(\)>): testa se o arquivo nomeado por este pathname abstrato é um arquivo oculto.

### Manipulando Arquivos e Diretórios

Vários métodos permitem criar arquivos e diretórios em um file system. A maioria deles é dependente do file system. Lembre-se de que esses métodos são legados. Você pode verificar a seção [Refatorando seu Código para Usar Path](<#/doc/tutorials/java-io/file-system/file-path>) para refatorar seu código e usar os métodos equivalentes da interface [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) e da classe [`Files`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Files.html>).

  * [`createNewFile()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#createNewFile\(\)>): tenta criar um novo arquivo a partir deste pathname. Esta criação falhará se este arquivo já existir. Este método retorna `true` se o arquivo foi criado com sucesso, e `false` se o arquivo já existe ou se um link simbólico com o mesmo nome existe. Se a criação do arquivo falhar por qualquer outro motivo, uma [`IOException`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/IOException.html>) é lançada.
  * [`delete()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#delete\(\)>): exclui o arquivo ou diretório denotado por este pathname abstrato. Se este pathname denotar um diretório, então o diretório deve estar vazio para ser excluído. Este método retorna `true` se o arquivo foi excluído com sucesso e `false` caso contrário. Você deve favorecer o uso do método [`Files.delete()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Files.html#delete\(java.nio.file.Path\)>) em vez deste, pois ele fornece mais informações caso a exclusão falhe. Observe que este método nunca lança uma [`IOException`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/IOException.html>) se o arquivo não existir.
  * [`mkdirs()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#mkdirs\(\)>) e [`mkdir()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#mkdir\(\)>): criam um diretório nomeado por este pathname abstrato. `mkdirs()` cria todos os diretórios intermediários, se necessário.
  * [`renameTo(file)`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#renameTo\(java.io.File\)>): renomeia o arquivo denotado por este pathname abstrato.

## Apresentando a Interface Path

A classe [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>), introduzida na versão Java SE 7, é um dos principais pontos de entrada do pacote [`java.nio.file`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/package-summary.html>). Se sua aplicação usa I/O de arquivos, você vai querer aprender sobre os poderosos recursos desta interface.

> Nota de Versão: Se você tem código pré-JDK7 que usa [`java.io.File`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html>), você ainda pode aproveitar a funcionalidade da interface [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) usando o método [`File.toPath()`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/io/File.html#toPath\(\)>). Veja a próxima seção para mais informações. Como o nome implica, a interface [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) é uma representação programática de um path no file system. Um objeto [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) contém o nome do arquivo e a lista de diretórios usados para construir o path, e é usado para examinar, localizar e manipular arquivos.

Uma instância de [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) reflete a plataforma subjacente. No Solaris OS, um [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) usa a sintaxe Solaris (`/home/joe/foo`) e no Microsoft Windows, um [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) usa a sintaxe Windows (`C:\home\joe\foo`). Um [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) não é independente do sistema. Você não pode comparar um [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) de um file system Solaris e esperar que ele corresponda a um [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) de um file system Windows, mesmo que a estrutura de diretórios seja idêntica e ambas as instâncias localizem o mesmo arquivo relativo.

O arquivo ou diretório correspondente ao [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) pode não existir. Você pode criar uma instância de [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>) e manipulá-la de várias maneiras: você pode anexar a ela, extrair partes dela, compará-la com outro path. No momento apropriado, você pode usar os métodos da classe [`Files`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Files.html>) para verificar a existência do arquivo correspondente ao [`Path`](<https://docs.oracle.com/en/java/javase/26/docs/api/java.base/java/nio/file/Path.html>), criar o arquivo, abri-lo, excluí-lo, alterar suas permissões e assim por diante.
